## Audio Requirements

Cisco Unity Connection supports the following audio formats for greeting recordings. All formats are 8kHz mono.

| Format | Bits | Avg Bytes/sec | Browser Playback | Notes |
|--------|------|---------------|------------------|-------|
| **PCM linear** | 16-bit | 16,000 | Yes | Lossless, largest file size |
| **G.711 mu-law** | 8-bit | 8,000 | Yes | Standard North American telephony codec — recommended |
| **G.711 a-law** | 8-bit | 8,000 | Yes | Standard European/international telephony codec |
| **G.729a** | — | 1,000 | **No** | Heavy compression, noticeable artifacts |
| **G.726** | 4-bit | 4,000 | **No** | ADPCM, half bandwidth |
| **GSM 06.10** | — | 1,625 | **No** | Mobile codec, slight artifacts |

**Which format is best?** G.711 mu-law is the sweet spot for North American systems. It's the native codec of the phone network — CUC delivers it to callers with zero transcoding. PCM linear is technically higher fidelity, but CUC must transcode it to G.711 for phone delivery anyway, so there's no audible benefit.

**For uploading custom greetings:** Record as 8kHz, 16-bit, mono WAV (PCM linear). CUC accepts this universally and converts to its internal storage format.

**Browser playback:** The wizard downloads greeting audio and embeds it in the reports for inline playback. Only PCM, G.711 mu-law, and G.711 a-law are playable in the browser. If your server stores greetings in G.729a, G.726, or GSM, the audio files will download but won't play in the report — they would need transcoding.

The wizard automatically detects the codec of each downloaded WAV file by reading the file header and reports a summary in the console output.

## Report Pages

All report pages share a **floating navigation pill** (top-right corner) for switching between views and a **dark/light mode toggle**. The nav pill is semi-transparent and becomes fully visible on hover.

Reports are generated into `reports/<SiteName>_<timestamp>/` and are fully self-contained — no internet connection is required to view them. All CSS, JavaScript, and audio files are included locally.

---

### Index (Home)

**File:** `index.html`

The landing page with cards linking to each report view. Each card has a brief description of what the page shows. This is the page that opens automatically after generation.

### Call Flow Explorer

**File:** `callflow.html`

An interactive step-by-step call flow tracer. This is the primary tool for understanding "what happens when someone calls."

#### Getting Started
1. Select a **schedule mode** (Standard, Off Hours, Holiday, or All) to see how routing changes by time of day.
2. Choose an **entry point** from the dropdown — these are routing rules that define how calls enter the system.
3. Click **menu keys** on each card to drill deeper into the call flow.

#### Flow Cards
Each card represents a call handler in the routing path:

- **Header:** Handler name, extension, schedule assignment, and admin link.
- **Audio rows:** Inline playback of greeting recordings with schedule badges and enabled/disabled status. The gear icon links to the greeting editor in CUC admin.
- **Extension dialing:** Shows which keys are unlocked for extension dialing and the digit timeout (how long CUC waits for additional digits).
- **Warning rows:** Red warning icons flag misconfigurations (see [Misconfiguration Warnings](#misconfiguration-warnings)).
- **Menu rows:** Each DTMF key option the caller can press. Click a row to expand the next handler in the flow.

#### Card Border Colors
| Color | Meaning |
|-------|---------|
| Green | Entry point (routing rule) |
| Blue | Normal call handler |
| Gold | Primary root handler (with glow) |
| Red | Dead end — caller has nowhere to go |

#### Deep Linking
The URL updates as you navigate through the flow. You can copy and share this URL — when opened, it reconstructs the exact path you were viewing. The **Copy Link** button (appears after 2+ steps) copies the deep link URL to your clipboard.

Deep link format: `callflow.html#s=standard&e=<entryId>&p=<label>~<nodeId>,<label>~<nodeId>,...`

#### Breadcrumb
The breadcrumb trail at the top shows your current path. Click any earlier step to jump back. The breadcrumb is sticky and stays visible as you scroll.

#### Loop Detection
If you click a menu option that leads to a handler already in your current path, the page scrolls to the existing card and flashes it instead of creating a duplicate.

### Graph View

**File:** `callhandler_map.html`

An interactive D3.js force-directed graph showing all handlers and their connections.

#### Layouts
Three layout modes are available via buttons:

| Layout | Description |
|--------|-------------|
| **Force** | Physics simulation — nodes repel each other, edges act as springs. Best for exploring organic relationships. |
| **Hierarchical** | Top-down tree layout based on BFS depth from routing rules. Best for understanding hierarchy. |
| **Radial** | Concentric circles based on depth. Same data as hierarchical but arranged in rings. |

#### Interacting with Nodes
- **Click** a node to select it. The sidebar shows full details and connections.
- **Drag** a node to reposition it. The node becomes **pinned** (red ring) and stays where you put it.
- **Double-click** a pinned node to unpin it and let the simulation place it.
- **Unpin All** button releases all pinned nodes.

#### Zoom and Pan
- **Mouse wheel** to zoom (range: 0.1x to 8x).
- **Click and drag** the background to pan.
- **Zoom In / Zoom Out** buttons for precise control.
- **Fit All** button auto-zooms to show every node with padding.

#### Node Highlighting
When a node is selected:
- **Outgoing connections** turn green with green labels.
- **Incoming connections** turn blue with blue labels.
- Unrelated nodes and edges fade to near-transparent.
- Click empty background to clear the selection.

#### Visibility Toggles
Three toggle buttons hide/show node classifications:
- **True Orphans** — isolated nodes with no connections.
- **Unreachable Subtrees** — have outgoing edges but nothing routes to them.
- **Dead Ends** — have incoming edges but callers can't go anywhere.

All are shown by default. Toggling off hides the nodes and their edges.

#### Sidebar Detail Panel
When a node is selected, the right sidebar shows:
- Name, extension, type, classification (color-coded).
- Schedule assignment.
- Extension dialing info (unlocked keys + digit timeout).
- Warnings (see [Misconfiguration Warnings](#misconfiguration-warnings)).
- Audio greetings with inline playback and enabled/disabled status.
- Incoming connections (count + list) — click any to navigate.
- Outgoing connections (count + list) — click any to navigate.
- Admin deep link to CUC.

#### Edge Labels
Edges are labeled with the routing action:
- `Key 1`, `Key 2`, etc. — DTMF menu key presses.
- `After:Standard`, `After:Off Hours`, etc. — after-greeting routing.
- `Xfer:Standard (ring 4x)` — supervised transfer with ring count.
- `Xfer:Standard (release)` — blind/release transfer.

#### Legend
The legend in the bottom-left shows all node classification colors and types.

### Handlers & Routing (Table Report)

**File:** `callhandler_report.html`

A comprehensive tabular report with call flow trees, handler details, and debug tools.

#### Schedule Mode Selector
Four buttons at the top filter the entire page by schedule context:
- **Standard** — shows routing as it works during business hours.
- **Off Hours** — shows off-hours routing.
- **Holiday** — shows holiday routing.
- **All** — shows every connection regardless of schedule (raw view).

Changing the schedule mode updates both the call flow trees and the handler table.

#### Call Flow Trees
Below the schedule selector, expandable BFS trees show the full routing path from each entry point. Each tree starts at a routing rule and follows menu keys, greetings, and transfers.

- Handler names are color-coded by type.
- Audio greetings are shown inline with playback controls.
- Visited nodes show "(see above)" to avoid infinite loops.
- Schedule badges appear on connections that are only active in certain schedules.
- **Copy** button exports the tree as plain text.

#### Handler Table
A searchable, filterable table of all handlers:

| Column | Content |
|--------|---------|
| Name | Handler name with admin link and system/primary badges |
| Extension | DtmfAccessId or "N/A" |
| Type | callhandler, interview, directory, routingrule, phone, action |
| Classification | Color-coded: root, normal, dead end, unreachable, orphan |
| Schedule/Conditions | Schedule name for handlers; routing conditions for rules |
| Incoming | Source handlers and edge labels |
| Outgoing | Target handlers and edge labels |
| Audio | Inline playback with schedule badges and enabled/disabled state |
| Object ID | CUC internal identifier |

- **Search bar** filters by name, extension, or type in real-time.
- **Classification dropdown** filters by node classification.
- Table headers are sticky when scrolling.

#### Debug Panel
A collapsible panel (bottom-right "Debug Tools" button) with three tools:

1. **Lookup Node** — search by name, extension, or Object ID. Returns full node data with all edges as formatted JSON.
2. **Find Problems** — generates a comprehensive problem report: orphans, dead ends, unreachable nodes, schedule gaps, and configuration warnings.
3. **Dump All Data** — exports the complete JSON dataset (all nodes, edges, schedules, holidays) for external analysis.

All outputs can be copied to clipboard.

### Schedules

**File:** `schedules.html`

Displays all business hour schedules and holiday schedules configured on the CUC server.

#### Business Hours Table
Shows each schedule's time blocks:
- Schedule name, active days, start time, end time, and active status.
- Schedules with no details show "All days, 12:00 AM – 11:59 PM" (always active).
- **Copy as Markdown** button for each table.

#### Holiday Schedules Table
Lists every holiday in each holiday schedule:
- Schedule name, holiday name, and date (or date range).

#### Holiday Calendar Audit
An automated audit banner appears above the holiday table if issues are detected. See [Holiday Calendar Audit](#holiday-calendar-audit) for details.

### Test Times

**File:** `test_times.html`

Computes recommended times to call the auto-attendant to verify each schedule state routes correctly.

#### Day Grouping
Days with identical schedule windows are grouped into ranges (e.g., "Monday – Thursday") to avoid repetition.

#### Test Time Logic
For each day group, the wizard recommends specific test times:

| Test Type | When | Example |
|-----------|------|---------|
| **Standard** | 5 minutes after business hours open | 8:05 AM |
| **Off Hours (gap)** | 5 minutes after a mid-day window closes | 12:05 PM |
| **Off Hours (after close)** | 5 minutes after business hours end | 5:05 PM |

Special cases:
- **24-hour schedule:** Single "Standard" test at noon.
- **No schedule:** Single "Off Hours" test — "No schedule active this day."

#### Holiday Testing Note
A note explains that holiday routing can only be tested by creating a temporary holiday entry in CUC for the current date, since holidays override both Standard and Off Hours.

#### Copy as Markdown
Each day group has a **Copy as Markdown** button that exports the table for pasting into documentation or tickets.

### Audit Results

A dedicated page that categorizes all findings from the call handler analysis into sections.

#### Summary Bar
The top of the page shows summary cards with counts for each category: total issues, handler warnings, holiday issues, classification concerns, audio issues, and system default greetings. Cards are color-coded: red for critical, orange for warnings, green for clean.

#### Misconfiguration Warnings
Lists every handler that has one or more misconfiguration warnings (supervised transfer, no timeout key, etc.), with CUC admin deep links and the handler's extension.

#### Holiday Calendar
Displays holiday audit findings with severity levels (CRITICAL, WARNING, OK).

#### Classification Concerns
Lists orphaned, unreachable, and dead-end handlers with explanations and admin links.

#### Audio Issues
Shows handlers with missing Standard greeting audio or non-browser-playable codecs.

#### System Default Greetings
Lists greetings using PlayWhat=1 (system default recording) instead of PlayWhat=2 (personal recording). This is informational — it flags greetings that may not have a custom recording uploaded.

## Node Classifications

Every node in the graph is classified based on its reachability from routing rules:

| Classification | Color | Meaning |
|----------------|-------|---------|
| **Root** | Green | Directly targeted by a routing rule — this is where calls enter the system. |
| **Normal** | Blue | Reachable from a root via menu keys, greetings, or transfers. |
| **True Orphan** | Gray | Completely isolated — no incoming or outgoing connections at all. |
| **Unreachable Subtree** | Orange | Has outgoing connections (it routes somewhere) but nothing routes *to* it. |
| **Dead End** | Red | Callers can reach it, but there's nowhere to go — no menu keys, no transfer, no after-greeting routing. |

Additional node types have their own colors:
| Type | Color | Description |
|------|-------|-------------|
| **Interview Handler** | Purple | Plays a series of interview questions. |
| **Phone Extension** | Teal | A phone number that calls are transferred to. Resolved to user/contact names when possible. |
| **Directory Handler** | Amber | Dial-by-name directory. |
| **Action Node** | Red | Terminal action like Hangup or Take Message. |

## Schedule Modes

CUC uses schedules to change call routing based on time of day. Edges in the graph are tagged with their schedule context:

| Schedule | When Active | Transfer Rule | Greeting |
|----------|-------------|---------------|----------|
| **Standard** | During business hours | Standard transfer rule | Standard greeting |
| **Off Hours** | Outside business hours | Off Hours transfer rule | Off Hours greeting |
| **Holiday** | During configured holidays | Holiday transfer rule (if set) | Holiday greeting |
| **Alternate** | Manually enabled by admin | Alternate transfer rule | Alternate greeting |
| **Always** | Always active | — | Menu key entries, routing rules |

The **Alternate** schedule is special — when enabled, it overrides both Standard and Off Hours. This is typically used for temporary closures (snow days, emergencies).

Schedule badges appear as colored pills throughout the reports to indicate which schedule context applies to each connection or greeting.

## Misconfiguration Warnings

The wizard automatically detects common auto-attendant misconfigurations. Warnings appear in the console during generation, on call flow cards, in the graph sidebar, and in the debug panel's "Find Problems" report.

| Warning | What It Means | Why It Matters |
|---------|---------------|----------------|
| **Alternate transfer rule enabled** | The Alternate transfer rule is active. | Overrides Standard and Off Hours transfer — all calls go to the alternate target until disabled. |
| **Alternate greeting enabled** | The Alternate greeting is active. | Callers hear the alternate greeting instead of Standard. Often left on accidentally after a temporary closure. |
| **Supervised transfer** | Transfer type is Supervised instead of Release. | For an auto-attendant, Release (blind transfer) is standard. Supervised transfer makes CUC monitor the call, which adds delay and can cause issues if the target doesn't answer. |
| **No timeout key (*)** | No action is configured for the * (timeout) key. | If a caller presses nothing, there's no fallback — they sit in silence. The * key fires when CUC times out waiting for input. |
| **After-greeting = Hangup** | The after-greeting action disconnects the caller. | After the greeting plays, the caller gets hung up on with no chance to press keys or be transferred. |
| **After-greeting = Take Message** | The after-greeting action prompts for voicemail. | On an auto-attendant (not a user mailbox), this usually means callers are unexpectedly asked to leave a message. |
| **After-message = Hangup** | After a caller records a message, they're disconnected. | No option to return to the menu or be transferred — the call just ends. |
| **Menu key = Take Message** | A DTMF key action is set to Take Message. | A menu key on an auto-attendant sending callers to voicemail is usually a misconfiguration. |
| **Key routes to itself** | A menu key targets the same handler it's on. | Creates a loop — pressing this key just replays the same handler. |
| **Circular routing** | Two handlers route to each other with no exit. | Callers bounce between two handlers with no way out. |
| **Record your message prompt** | The "Record at the tone" prompt is enabled. | On an auto-attendant handler, this prompt is usually unwanted — callers aren't expecting to leave a voicemail. |
| **Caller input disabled** | IgnoreDigits is true on a greeting that has active menu keys. | DTMF keys are ignored during greeting playback — callers can't press anything until the greeting finishes. This can be intentional (force callers to listen) but is often a misconfiguration. |

## Holiday Calendar Audit

The wizard audits holiday schedules for staleness every time reports are generated or the `schedules` command is run.

| Level | Condition | What To Do |
|-------|-----------|------------|
| **CRITICAL** | No holidays configured for the current year. | Add holidays for this year immediately — holiday routing won't work. |
| **CRITICAL** | It's December and no holidays exist for next year. | The holiday calendar needs updating before year-end. |
| **WARNING** | Holidays from a previous year are still present. | Clean up old entries. They're harmless but indicate the calendar may not be actively maintained. |
| **OK** | Current year holidays are present and (if December) next year is covered. | No action needed. |

The audit appears as a color-coded banner on the **Schedules** page and prints to the console during generation.

## Audio Codec Detection

When greeting audio files are downloaded during report generation, the wizard reads each WAV file's header to identify the codec. A summary is printed to the console:

```
  Codecs: G.711 mu-law (6), PCM linear (2)
```

If any files use a codec that browsers cannot play (G.729a, G.726, or GSM 06.10), the audio entry is flagged. See [Audio Requirements](#audio-requirements) for the full codec compatibility table.

## CUC Admin Deep Links

Throughout the reports, small link icons open the corresponding object directly in the CUC administration interface (Cisco Unity Connection Administration).

| Object | URL Pattern |
|--------|-------------|
| Call Handler | `https://<host>/cuadmin/callhandler.do?op=read&objectId=<id>` |
| Interview Handler | `https://<host>/cuadmin/interviewhandler.do?op=read&objectId=<id>` |
| Directory Handler | `https://<host>/cuadmin/directoryhandler.do?op=read&objectId=<id>` |
| Routing Rule | `https://<host>/cuadmin/routingrule.do?op=read&objectId=<id>` |
| Greeting Editor | `https://<host>/cuadmin/greeting.do?op=readCallhandler&objectId=<handlerId>&greetingType=<type>` |

Deep links only appear when the `--host` parameter was provided during generation. You must be logged into CUC admin in the same browser for these links to work.

## Dark / Light Mode

All report pages include a theme toggle in the floating navigation pill (top-right). Click the sun/moon icon to switch.

- **Dark mode** (default): Dark blue/navy backgrounds, light text. Optimized for extended reading.
- **Light mode**: White/light gray backgrounds, dark text. Better for printing or bright environments.

The selected theme is persisted in your browser's localStorage (`chw-theme` key) and applies across all pages.

## CLI Commands

The wizard includes several CLI commands beyond report generation:

### generate (default)

```bash
python callhandler_wizard.py --host https://... --user admin
```

Full report generation. Fetches all data, builds the routing graph, downloads audio, detects misconfigurations, runs the holiday audit, and generates all HTML pages.

### handler

```bash
python callhandler_wizard.py --host https://... --user admin handler "Opening Greeting"
python callhandler_wizard.py --host https://... --user admin handler 2000
python callhandler_wizard.py --host https://... --user admin handler <objectId> --raw
```

Look up a single handler by name, extension, or Object ID. Shows transfer rules, greetings, and menu entries. Add `--raw` for full JSON dump.

### schedules

```bash
python callhandler_wizard.py --host https://... --user admin schedules
```

Lists all business hour and holiday schedules with their time blocks. Includes the holiday calendar audit.

### orphans

```bash
python callhandler_wizard.py --host https://... --user admin orphans
```

Finds orphaned, unreachable, and dead-end handlers. Includes per-schedule reachability analysis — a handler might be reachable during Standard hours but orphaned during Off Hours.

### query

```bash
python callhandler_wizard.py --host https://... --user admin query /vmrest/handlers/callhandlers
```

Hit any CUPI API path directly and dump the raw JSON response. Useful for exploring the API.

### probe

```bash
python callhandler_wizard.py --host https://... --user admin probe
```

Tests 40+ CUPI endpoints to map what's available on the server. Shows available, unavailable, and errored endpoints with record counts.

### audio

```bash
python callhandler_wizard.py --host https://... --user admin audio
```

HEAD-checks every handler's greeting audio URLs to verify which greetings have uploaded recordings without downloading the files.
